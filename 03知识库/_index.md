# 03知识库

> 跨项目沉淀的知识资产。与 `01-通用规则/`（方法层）的区别：这里放的是**内容**（案例、政策、研究），那里放的是**方法**（怎么干）。

## 目录结构

| 子目录 | 放什么 | 什么时候建 |
|--------|--------|-----------|
| **工作案例库/** | 过往项目的 PRD、方案、复盘，按领域分类 | 项目结项后沉淀 |
| **行业研究/** | 行业趋势、市场规模、产业链分析 | 主动研究时 |
| **竞品分析/** | 竞品功能对比、产品拆解 | 竞品调研时 |
| **政策法规/** | 行业相关政策、法规、标准（如政务相关规范） | 政策更新时 |
| **技术学习/** | 技术栈学习笔记、架构理解 | 技术调研时 |
| **读书笔记/** | 产品/管理/技术书籍读书笔记 | 读书时 |

## 治理原则

- **先有项目后有案例**：不要空建案例库，跑完真实项目再沉淀
- **一个领域积累 3 个以上案例后**，提炼共性为方法论笔记（放到 `01-通用规则/方法论/`）
- **入库要写索引**：每个子目录建 `_index.md`，让别人能找到你的案例
- 详细治理规范见 `01-通用规则/项目工作流规范.md` 第十章（案例沉淀与知识库治理）

## 团队共享场景

如果你用乐享/飞书等云知识库做团队共享，建议：
- **云知识库**放业务知识（闽政通业务规则、客户资料、行业政策）—— 人读、多人共建
- **本地这里**放与 AI 执行紧密关联的内容 —— 被 skills 引用
- 两者职责不同，不要混。详见团队架构说明。
