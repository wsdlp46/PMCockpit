# 项目模板使用说明

> 这是一个**空的项目骨架模板**。复制它、改名、填内容，就是一个完整的项目。

## 怎么用（两种方式）

### 方式 A：复制模板（推荐，结构最完整）

```bash
cd 02当前项目
cp -R _模板项目 你的项目名
```

然后把里面所有 `<占位符>` 替换成真实内容，把 `_模板项目` 相关字样改成你的项目名。

需要替换的关键文件：
- `00-项目总览.md` — 填项目信息、产品架构、角色端侧
- `_index.md` — 项目名替换
- `01-项目基线/关键决策.md` — 开始登记决策
- `01-项目基线/需求池.md` — 开始记需求
- `项目工作流约定.md` — 项目专属约定

### 方式 B：让 AI 建（更省事）

对 AI 说"开始新版本"，AI 会按 project-iteration skill 的流程，引导你确认版本号并自动建出版本目录骨架（01-原始需求 到 06-变更记录）。

但注意：skill 只建**版本目录**（02-迭代/01-V1.0.0/），不建**项目根骨架**（00-项目总览、01-项目基线）。项目根骨架建议用方式 A 复制本模板。

## 推荐流程

1. 用方式 A 复制模板，建出项目根骨架
2. 填好 00-项目总览（至少项目名、角色端侧、当前阶段）
3. 在 01-项目基线/关键决策.md 登记已知决策
4. 对 AI 说"开始新版本"，建第一个版本的迭代目录
5. 开始写需求、画原型

## 目录结构速查

```
<项目名>/
├── 00-项目总览.md          ← 项目仪表盘（人读）
├── _index.md               ← 项目索引（AI 读）
├── 项目工作流约定.md        ← 项目专属补充
├── 01-项目基线/            ← 跨版本累积资产
│   ├── 关键决策.md          ← #D0XX 决策登记
│   ├── 需求池.md            ← 需求清单
│   ├── 业务规则/            ← 业务规则详述
│   ├── 会议记录/            ← 时间线 + 原始纪要
│   ├── 架构图/              ← 产品/系统架构图
│   ├── 设计规范/            ← 项目专属设计规范
│   ├── 法律协议/            ← 合同、协议
│   ├── 项目工具/            ← 上线清单、排期
│   └── _source/             ← 基线二进制原件
├── 02-迭代/                ← 版本目录（skill 自动建）
│   └── _index.md            ← 版本索引
├── 03-对外材料/            ← 方案书、汇报
├── _source/                 ← 临时二进制原件
└── 归档/                    ← 已废弃内容
```
