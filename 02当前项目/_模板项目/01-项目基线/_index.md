# 01-项目基线

> 跨版本累积的资产。一个项目跑多个版本，这里的东西是所有版本共享的基础。

## 目录

| 子目录 | 放什么 |
|--------|--------|
| `关键决策.md` | 设计决策登记（#D0XX 编号），项目"宪法" |
| `需求池.md` | 所有已知需求清单，迭代时从这里挑 |
| `业务规则/` | 业务流程规则、计算规则、合规要求等详述 |
| `会议记录/` | 会议纪要时间线 + 原始纪要（转写、记录） |
| `架构图/` | 产品架构图、系统架构图、流程图 |
| `设计规范/` | 本项目专属的设计规范、品牌色、Logo |
| `法律协议/` | 注册协议、隐私协议、合作合同等 |
| `项目工具/` | 上线清单、项目管理工具模板 |
| `_source/` | 基线相关二进制原件（PDF、Word）+ 转换 MD |

## 维护规则

- 关键决策、需求池：有新内容随时追加，不删旧条目
- 会议记录：每次会议后 24 小时内入库，更新时间线
- 业务规则：业务方确认后入库，标注来源和确认时间
- 架构图：版本变更时更新，旧版移入 `归档/`
