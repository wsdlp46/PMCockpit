# 示例项目

这里是一个最小可运行的项目骨架 `demo-management-system/`，演示 PMCockpit 的标准项目结构。

## 怎么用

**方式一：照着抄一个你自己的项目**

复制 `demo-management-system/` 目录，改名为你的项目（如 `我的订单系统/`），把里面所有 `<占位符>` 替换成真实内容，然后对 AI 说"开始新版本"。

**方式二：直接在这个 demo 上试**

```bash
cd examples/demo-management-system
# 对 AI 说："开始新版本 V1.0.0"
```

AI 会按 project-iteration skill 的流程，在这个项目下建出 `02-迭代/01-V1.0.0/` 的完整目录骨架（01-原始需求 到 06-变更记录），然后你就可以开始写需求、画原型了。

## 目录结构对照

```
demo-management-system/
├── 00-项目总览.md          ← 项目仪表盘（角色、端侧、迭代状态）
├── 01-项目基线/
│   ├── 关键决策.md          ← 跨版本累积的设计决策（#D0XX 编号）
│   └── 会议记录/            ← 外部输入归档
└── 02-迭代/
    └── _index.md           ← 版本索引（AI 第一眼看这里）
```

> 注意：这个 demo 不含 `03知识库/`（工作案例库）和 `03-对外材料/`，它们是可选的。知识库建议等项目跑完一两个版本、有案例可沉淀时再建，参考 `01-通用规则/项目知识库搭建指南.md`。
