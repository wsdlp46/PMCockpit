# 工作区主页

> 这是你的产品经理 AI 工作区。基于 PMCockpit 脚手架搭建。

## 快速入口

- [01-通用规则/](01-通用规则/) — 方法层：项目工作流规范、模板、方法论、检查清单、设计规范。被 AI skills 自动引用。
- [02当前项目/](02当前项目/) — 进行中的项目。每个项目一个子目录。
- [03知识库/](03知识库/) — 知识沉淀：工作案例库、行业研究、竞品分析、政策法规。
- [04历史项目/](04历史项目/) — 已结项归档。
- [06-对外材料/](06-对外材料/) — 方案书、投标文档、对外介绍。

## 怎么开始

1. 确认已运行 `bash .ai/install.sh`（装好 skills）
2. 打开 AI 工具，工作目录指到本仓库根
3. 在 `02当前项目/` 下建你的项目（对 AI 说"开始新版本"）
4. 开始写需求、画原型

详细操作见 [01-通用规则/助理上手指南.md](01-通用规则/助理上手指南.md)

## 工作区结构

```
工作区根/
├── 00主页.md              ← 你在这里
├── AGENTS.md              ← AI 工作区规则（写作纪律、硬约束）
├── .ai/                   ← skills + agents + install.sh
├── 01-通用规则/            ← 【通用层】方法层（AI 引用）：规范、模板、方法论、检查清单、设计规范
├── 02当前项目/            ← 你的项目（每个项目一个子目录）
├── 03知识库/              ← 【通用层】知识沉淀：工作案例库、行业研究、竞品分析、政策法规
├── 04历史项目/            ← 归档
├── 06-对外材料/           ← 对外文档
└── _附件/                 ← 二进制文件
```

## 项目内部结构（四层）

每个项目内部采用四层结构（通用层在工作区根，见上）。详细规则见 [01-通用规则/项目工作流规范.md](01-通用规则/项目工作流规范.md) 第一章。

```
项目名/                    ← 【项目层】
├── 00-项目总览.md          ← 项目仪表盘
├── _index.md               ← AI 索引
├── 01-项目基线/            ← 跨版本累积资产（关键决策、需求池、业务规则、设计规范、架构图、会议记录）
├── 02-迭代/                ← 【迭代层】所有版本的容器
│   └── 0X-V1.0.X/          ← 【版本内目录层】单个版本，完整自包含
│       ├── 01-原始需求/    ← 本版本外部输入
│       ├── 02-需求文档/    ← PRD + 功能清单
│       ├── 03-规格文档/    ← 原型设计规格
│       ├── 04-原型/        ← HTML 原型 + _scaffold/
│       ├── 05-TAPD上传规划/
│       └── 06-变更记录/    ← 评审纪要、变更清单
├── 03-对外材料/            ← 汇报 PPT、产品介绍
└── _source/、归档/
```

> 通用层（`01-通用规则/`、`03知识库/`）是所有项目的共享输入；项目层以下自包含，结项后整体移到 `04历史项目/`。
