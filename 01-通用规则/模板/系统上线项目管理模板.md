# 系统上线项目管理模板

> 来源：<行业>私有化部署范本（2021）→ 实战调整（2026-06）
> 入库日期：2026-06-14
> 触发词："整理上线计划"、"上线前还有哪些事"、"帮我列项目排期"、"系统上线需要准备什么"
>
> 本模板核心结构源于真实项目的实战调整，原始停车范本详见同目录 Excel 文件。

---

## 通用框架（三阶段）

| 阶段 | 核心问题 | 典型耗时 |
|------|------|:---:|
| 前期准备 | 外部条件是否就绪（公司资质、外部申请、规则决策） | 2-4周 |
| 项目执行 | 内部是否能交付（开发、配置、测试） | 3-5周 |
| 上线准备 | 运营是否跟得上（人员培训、物料准备、上线执行） | 1-2周 |

> 数据割接类项目（系统替换场景）在"项目执行"末尾增加割接任务，在"上线准备"增加割接当日清单。

---

## 列定义（9列）

| 列名 | 说明 | 示例/可选值 |
|------|------|------|
| 阶段 | 三阶段之一 | 前期准备 / 项目执行 / 上线准备 |
| 板块 | 执行主体或场景分组 | 公司主体 / 平台开发 / 运营准备…… |
| 事项 | 任务名称 | 简洁明确，不编序号 |
| 优先级 | P0 > P1 > P2 | P0=不完成则阻塞 / P1=应完成 / P2=可后续 |
| 需要执行的动作 | 任务说明 | 可多行；同类子规则合并为一个单元格 |
| 状态 | 初始值预填"待开始" | 待开始 / 进行中 / 已完成 / 已延期 |
| 建议负责人 | 具体责任人 | 真实姓名或角色简称 |
| 计划开始 | 计划开始日期 | YYYY-MM-DD |
| 计划完成 | 计划完成日期 | YYYY-MM-DD |

> 对比原始停车模板（13列）：去掉了编号、项目类型、备注、场景四列，列序改为优先级紧接事项名。

## 板块分组原则

按**执行逻辑**而非功能域分组：

```
前期准备 → 谁能做什么？
  ├── 公司主体       → 营业执照/Logo/商标（公司存续类）
  ├── 外部申请       → 域名/服务器/小程序/运营商/支付/短信（向第三方申请）
  └── 规则制定       → 协议/代理规则/定价/数据安全（内部决策类）

项目执行 → 执行顺序是什么？
  ├── 平台开发       → 编码和部署
  ├── 平台配置       → 系统参数设置
  ├── 运营配置       → 业务规则配置
  └── 平台测试       → 各维度测试验收

上线准备 → 运营能跟上吗？
  ├── 运营准备       → 签约/培训/素材/文档
  └── 上线           → 公告/验证/监控/复盘
```

## 输出规则（优先级排序）

1. **先确定板块**：按"公司主体→外部申请→规则制定→平台开发→平台配置→运营配置→平台测试→运营准备→上线"顺序，哪个板块有事项就列哪个
2. **只列Now阶段**：远期跟踪项（如AI牌照动态）不列入本表，另表管理
3. **优先级紧跟事项名**：P0/P1/P2要一眼看到
4. **不编序号**：事项名直接描述，不用B1/L1/D1这类机械编号
5. **同类合并**：多条子规则可合并为一个多行格（如"代理商核心规则"）
6. **状态预填**：初始值统一"待开始"
7. **列序固定**：阶段→板块→事项→优先级→需要执行的动作→状态→建议负责人→计划开始→计划完成
8. **优先腾讯在线表格**：合并单元格+首行冻结，方便多人协作
