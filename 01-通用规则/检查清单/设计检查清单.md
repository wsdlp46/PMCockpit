# 设计检查清单

> 配套文档：[Web应用设计规范](../设计规范/Web应用设计规范.md)

---

## 交互检查

- [ ] 导航层级是否 ≤ 3？超过时是否使用筛选器/树状图？
- [ ] 同一页面导航类型是否 ≤ 3种？
- [ ] 核心按钮是否 ≤ 2个？按钮总数是否 ≤ 5个？
- [ ] 表单是否支持默认填写和关联信息推荐？
- [ ] 页面跳转是否遵循"关联度高→当前页，不同业务→新开页"原则？
- [ ] 信息反馈是否选择了正确的类型（Toast/警示/弹窗）？

## UI检查

- [ ] 设计尺寸是否基于 1440×810？
- [ ] 是否采用24栅格系统？
- [ ] 页面边距是否为24px？
- [ ] 输入框高度是否为40px？行内编辑为32px？
- [ ] 按钮圆角是否为2px？
- [ ] 表格行高是否为40px？斑马线色值是否正确？
- [ ] 弹窗遮罩透明度是否为60%？侧滑窗蒙版透明度是否为45%？
- [ ] 字阶是否控制在3~5种？
- [ ] 颜色种类是否受限？是否突出主体信息？
- [ ] Loading是否符合等待时间分级处理规范？

## 原型检查

> 详见 [原型设计规范](../方法论/原型设计规范.md)

- [ ] 文件命名是否规范（`[项目名]-[模块名]-V[版本号].rp`）？
- [ ] 页面是否包含：封面页→修订记录→功能清单/导航→详细功能页？
- [ ] 交互标注是否覆盖跳转关系、数据来源、异常状态？
- [ ] 通用组件是否复用组件库，项目内保持一致？
