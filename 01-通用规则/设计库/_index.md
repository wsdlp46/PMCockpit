# 设计库

> HTML原型生成时的视觉基线参数库。不是品牌规范（品牌规范见 `01-通用规则/设计规范/政务APP设计规范.md`），而是每个端态可直接引用到原型代码中的CSS参数集合。跨项目共享的设计约定见 `设计约定.md`。
>
> 最后更新：2026-06-18

---

## 一、三端分类

| 端态 | 文件 | 适用场景 |
|------|------|------|
| **web后台** | `web后台-design.md` | B端管理后台、运营后台、配置平台 |
| **web门户** | `web门户-design.md` | 对外门户首页、数据资产门户、聚合页 |
| **手机端** | `手机端-design.md` | H5轻应用、微信小程序、移动端页面（政务/强调色风） |
| **手机端-极简黑白** | `手机端-极简黑白-design.md` | 移动端电商、钱包/支付、订单、个人中心（黑白极简风） |

## 二、选型流程

项目原型设计启动后，在 `proto-spec-generator` 的端侧确认环节，按以下决策链走：

```
1. 项目含哪些端？
   ├── 后台管理页面 → 加载 web后台-design.md
   ├── 对外门户/首页 → 加载 web门户-design.md
   ├── H5/小程序页面 → 加载 手机端-design.md
   └── 多端混合 → 各自加载，端际独立

2. 项目有自定义主题吗？
   ├── 有 → 用项目主色覆盖设计中默认的色值（其余参数保留）
   └── 无 → 全部使用默认值

3. 参照已有案例
   - web后台可参照：<项目名> 和 <项目名> 中的管理后台原型
   - 手机端可参照：<项目名> 中的H5原型
   - 手机端-极简黑白可参照：My Orders / E-Wallet / Profile 系列原型（黑实心按钮 + 白卡片 + 中圆角）
```

## 三、与已有规范的关系

| 资源 | 关系 |
|------|------|
| `01-通用规则/设计规范/政务APP设计规范.md` | 品牌级规范，含色值/字号/组件/适老化。设计库引用其色值和字号作为默认值，但不重复抄写 |
| `proto-spec-generator` skill | 该技能的"UI风格确认"步骤引用本库进行参数加载 |
| `prototype-html` skill | 该技能生成HTML原型时，先读取对应的design.md获取CSS基线值 |

---

## 四、维护规则

- 每个design.md只放**生成HTML原型直接需要的CSS参数**，不放设计理论
- 参数以表格为主，尽量减少段落描述
- 项目复盘发现有价值的参数模式 → 更新对应design.md
- 新增端态类型 → 在本目录新建文件 + 更新本README
