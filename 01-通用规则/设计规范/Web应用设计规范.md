## 1 用户体验设计定义

### 1.1 核心内涵

用户体验设计涵盖以下方面：

1. 了解产品目的和用户目的，并以此指导设计
2. 为用户提供清晰易理解的业务内容、有序整理的信息结构、便于信息检索的导航、信息主次分明的界面
3. 为用户提供有用的功能、流畅的交互操作、易于理解的信息布局、美观大方的界面

### 1.2 信息产品特性

中后台信息管理系统通常功能繁多、信息量巨大，"信息产品"特性占比偏重。因此，用户体验设计应主要从**信息结构**、**信息设计**等方面考虑如何优化。

### 1.3 局限性

用户体验设计主要从结构层、框架层、表现层去优化产品。完整的产品设计还需从战略层（目的和用户需求）、范围层（功能、业务流程）等多方面进行规范和优化。

用户体验优化是持续进行的过程，应及时收集归纳用户反馈，不断打磨产品。

## 2 设计原则

B 端后台产品通常涉及业务信息繁多、功能复杂、信息量巨大，设计应主要从信息结构和信息设计角度优化，遵循以下五项核心原则：

| 原则 | 说明 |
|------|------|
| **基于角色** | 根据不同角色的业务需求、操作习惯、使用情景等因素，在合适的步骤和时间提供与角色对应的信息与功能 |
| **清晰** | 消除界面中的信息杂质和歧义，让用户有效地看到信息、理解信息、处理信息 |
| **一致** | 对相似业务场景建立相似的解决方案，提高用户对系统的熟悉度，建立统一的直觉和心智模型 |
| **高效** | 简化和优化工作流程，通过对用户需求的预判，为办公提供更智能、更快速的业务解决方案 |
| **美观** | 用科学的设计方法提升美学和观感，契合智能化办公场景下专注而严谨的审美诉求 |

## 3 视觉基础

### 3.1 色彩系统

颜色体系分为四部分：**品牌色**、**中性色**、**功能色**、**数据图表色**。

#### 3.1.1 品牌色

品牌色用于主按钮、链接、ICON、部分控件等，是日常业务中经常使用的颜色。

| 色彩 | 色值 | 用途 |
|------|------|------|
| 品牌蓝（主色） | `#1E63B2` | 主按钮、链接、选中态、图标、控件 |

#### 3.1.2 中性色

灰色系（含蓝灰）用于字体、背景、边框、分割线等。**中性色固定不变，不作定制**。

**文字色级**：

| 用途 | 色值 |
|------|------|
| 主要文字 | `#1F1F1F` |
| 次要文字 | `#666666` |
| 辅助文字 | `#999999` |
| 禁用文字 | `#CCCCCC` |

**背景与边框**：

| 用途 | 色值 |
|------|------|
| 边框/分割线 | `#E5E5E5` |
| 页面背景 | `#F5F5F5` |

**表格行色**：

| 名称 | 色值 | 用途 |
|------|------|------|
| 列表头部 | `#F6F6F6` | 表格表头背景 |
| 浅色行 | `#FFFFFF` | 表格正文浅色行 |
| 深色行 | `#FBFBFC` | 表格正文深色行（斑马纹） |
| 悬停态 | `#1E63B2` 10% 透明度 | 表格行悬停 |
| 选中态 | `#1E63B2` 10% 透明度 | 表格行选中 |

#### 3.1.3 功能色

功能色代表明确的信息和状态，需遵守用户对色彩的基本认知，**一套产品体系下功能色应保持一致**。

| 状态 | 色值 | 说明 |
|------|------|------|
| 成功 | `#52C41A` | 操作成功提示 |
| 警告 | `#FAAD14` | 警示信息 |
| 错误 | `#F5222D` | 错误警告 |
| 信息 | `#1890FF` | 信息提示 |

#### 3.1.4 状态用色

用于业务状态的视觉标识：

| 状态 | 标识方式 | 说明 |
|------|----------|------|
| 未送审/待处理 | 黄色圆点 + 状态文案 | 待提交状态 |
| 已送审/已完成 | 绿色圆点 + 状态文案 | 已提交状态 |
| 退回/驳回 | 红色圆点 + 状态文案 | 被退回状态 |
| 作废/无效 | 灰色圆点 + 状态文案 | 已作废状态 |

#### 3.1.5 遮罩

| 场景 | 色值 | 透明度 |
|------|------|--------|
| 弹窗遮罩 | `#000000` | 60% |
| 侧滑窗遮罩 | `#000000` | 45% |

### 3.2 字体系统

#### 3.2.1 字体选择

- macOS 系统优先使用系统默认字体渲染
- Windows 系统按以下兼容性顺序选择字体：系统默认中文字体栈（微软雅黑 → 宋体）
- 英文及数字推荐 `Inter` 作为首选字体

#### 3.2.2 字号及行高

基本规则：行高为字号的 1.6 倍。正文基础字号为 14px，对应行高 22px。

| 层级 | 字号 | 行高 | 用途 |
|------|------|------|------|
| 特大 | ≥24px | auto | 页面大标题 |
| 大号 | 20px | 32px | 模块标题 |
| 中号 | 16px | 26px | 卡片标题、导航 |
| **正文** | **14px** | **22px** | 默认正文、表单、列表 |
| 辅助 | 12px | 20px | 说明文字、标签、辅助信息 |

> 系统中字阶选择应控制在 3-5 种之间，保持克制。

#### 3.2.3 行宽

- 正文一行字数不超过 50 个字符
- 首页标题文字 8-20 字为佳
- 基础行宽 = 字号 × 30（基于 Robert Bringhurst 方法），即 14px × 30 = **420px**
- 较适宜的行宽为 40-70 个字符
- 正文左右至少保留 16px 留白

#### 3.2.4 表格内文字

| 元素 | 行高 | 字号 |
|------|------|------|
| 表头行 | 40px | 14px |
| 内容行 | 32px | 14px |
| 行内输入框 | 32px | 14px |

### 3.3 间距系统

#### 3.3.1 基础栅格

以 **8px** 作为原子单位的栅格系统，所有间距取 8 的倍数。

#### 3.3.2 页面级间距

| 元素 | 间距值 |
|------|--------|
| 页面统一边距 | **24px** |
| 列表之间间距 | 64px |
| 列与列间距 | 16px |

#### 3.3.3 组件级间距

| 元素 | 间距值 |
|------|--------|
| 标题文字与输入框距离 | 8px |
| 输入框内文字与边框距离 | 16px |
| 上下排列输入框间距 | 24px（可调整为 16px） |
| 有错误提示时竖向距离 | 以提示内容与下方输入框计算，或横向显示在输入框右侧 |

### 3.4 图标规范

图标分为三种形态：

| 类型 | 用途 |
|------|------|
| **线性图标** | 导航按钮 icon、独立 icon 按钮常态/hover 态、进度圈 |
| **填充图标** | 树状导航按钮、标签操作按钮、独立 icon 按钮选中态、提示状态 |
| **彩色图标** | 有色彩需求的特殊 icon（插画类按钮、缺省页面等） |

## 4 布局规范

### 4.1 页面分层

页面采用三层结构：

| 层级 | 说明 |
|------|------|
| **L0 — 页面层** | 主体页面内容 |
| **L1 — 侧滑窗层**（z-index: 1000） | 侧滑面板，单层最多一个 |
| **L2 — 弹窗层**（z-index: 1050） | 模态弹窗，单层最多一个 |
| **L3 — 确认弹窗层**（z-index: 1100） | 确认操作弹窗，蒙版不穿透 |

### 4.2 栅格系统

#### 4.2.1 设计基准

以 **1440 × 810** 为设计基准分辨率，向上适配或向下适配误差较小。

#### 4.2.2 24 栅格系统

采用灵活的 24 栅格系统，遵循"左定右动"原则：左侧菜单栏固定宽度，右侧内容区进行栅格。

```
总宽度 = 左侧菜单栏 + (列宽 × 列数 + 水槽 × 水槽数 + 左右间距)
1440   = 224       + (24 × 24  + 24 × 23   + 44 × 2)
```

| 参数 | 值 |
|------|-----|
| 左侧菜单栏固定宽度 | **224px** |
| 右侧栅格数 | **24 列** |
| 单列宽度 | **24px** |
| 水槽宽度 | **24px** |
| 左右间距 | **44px** |

### 4.3 页面框架

页面由以下区域组成：

| 区域 | 位置 | 内容 |
|------|------|------|
| **系统栏** | 页面顶部固定 | 系统名称、账号登录等系统全局功能 |
| **头部导航区** | 系统栏下方 | 当前业务域的子业务标签、业务域切换器、工具栏 |
| **内容区域（头部）** | 内容区顶部 | 状态标签、内容区域工具栏 |
| **内容区域（正文）** | 内容区中部 | 表格、表单内容和工具栏 |

## 5 导航与信息架构

### 5.1 导航类型

系统提供四种导航形式：

| 导航类型 | 位置 | 功能 | 特性 |
|----------|------|------|------|
| **全局导航** | 页面顶部固定 | 首页、模块、业务间快速切换 | 始终固定，不可隐藏 |
| **一级导航** | 页面左侧固定 | 各业务域的子业务入口 | 可折叠隐藏 |
| **状态导航** | 信息面板左上角 | 子业务状态展示 | 按业务操作需求展示 |
| **树形导航** | 列表内容区左侧 | 层级数据展示与筛选 | 可收起、可展开、可拖拽 |

### 5.2 导航设计原则

1. 同一页面中最多使用**三种**导航
2. 信息层级 ≤ 3 时，使用优先级：侧边导航 > 一级标签页 > 二级标签页/锚点
3. 信息层级 > 3 时，可使用筛选器、树状图、导航/标签扩展，优先级：筛选器 > 导航/标签扩展 > 树状图
4. 建议提供按功能分类和按业务角色分类两种导航方式
5. 导航的信息闭环设计——区别于"绝对恪守从属逻辑"的树型结构，采用以页面为基线的复杂架构，通过信息关联将内容按主题链合成可回到原点的闭环，导航方式从"列表→业务→列表→业务"变为"业务→业务"

### 5.3 页面跳转原则

页面跳转从**关联度**（与主流程步骤或主界面内容的关联度）和**容量**（展示信息量的大小）两个维度定义：

| 跳转方式 | 适用场景 | 说明 |
|----------|----------|------|
| **当前页** | 关联度高、对主界面内容有影响的操作 | 在当前页面直接完成 |
| **下钻页** | 关联度中等、需要展开查看详情 | 在当前上下文内展开 |
| **新开页** | 属于另一个业务的信息 | 在一级导航新开标签页打开 |

## 6 组件规范

### 6.1 按钮

#### 6.1.1 通用规则

| 属性 | 规范 |
|------|------|
| 圆角 | **2px** |
| 文字 | 使用动词或动词短语，精准明确告知操作，禁用态需提供操作提示 |
| 术语 | 同级页面用"返回"，弹出页面用"关闭"，与"确定/保存"并存时用"取消" |
| 填充按钮间距 | 16px（组间）/ 8px（组内） |

#### 6.1.2 按钮尺寸

| 尺寸 | 高度 | 使用场景 |
|------|------|----------|
| 大号 | **36px** | 页面主操作 |
| 中号 | **32px** | 常用操作 |
| 小号 | **24px** | 表格内操作、辅助操作 |

> 按钮尺寸按 4 的倍数设定。

#### 6.1.3 布局规则

1. 页面核心指令按钮放在一级页面
2. 核心指令按钮 ≤ 2 个，视觉优先级最高，功能和页面标签名称呼应
3. 次级指令按钮 1-3 个
4. 一级页面指令按钮总数 ≤ 5 个（7±2 原则），其余按钮以下拉菜单形式收起
5. 当只有一个业务类型且操作按钮都全局控制时，按钮放在顶部

#### 6.1.4 按钮内有子按钮

支持按钮内嵌子按钮的复合结构。

### 6.2 输入框

#### 6.2.1 基础规范

| 属性 | 规范 |
|------|------|
| 高度 | **40px**（页面输入框）；**32px**（表内输入框） |
| 圆角 | **2px** |
| 内边距 | 文字距边框 **16px** |
| 输入框内字号 | **14px** |
| 标题位置 | 左对齐，标题与输入框间距 **8px** |

#### 6.2.2 交互状态

| 状态 | 说明 |
|------|------|
| 默认 | 正常边框，无特殊样式 |
| 输入结果 | 填充后的正常显示 |
| 提示错误 | 红色边框 + 下方红色错误提示文字 |
| 禁用 | 灰色背景，不可交互 |
| 获取焦点 | 品牌蓝色边框高亮 |

#### 6.2.3 必填提示

输入框标题前方带红色 `*` 号的为必填项。漏填或错填时，输入框下方显示错误提示。

#### 6.2.4 自适应规则

| 页面宽度 | 列数 | 说明 |
|----------|------|------|
| < 720px | 1 列 | 单列布局 |
| 721–980px | 2 列 | 双列布局 |
| 981–1280px | 3 列 | 三列布局 |
| > 1280px | 3 列及以上 | 居中页面 |

- 等宽输入框：按页面尺寸列数图进行自适应
- 长短输入框 1：最多 3 列，超过 1281px 时 3 列并居中页面；需要较宽输入框时可调整宽度与上下分布对齐
- 长短输入框 2：列数不受页面宽度限制；超出时可加滚动条；列间距 16px，宽度以 8 为最小倍数

### 6.3 选择器

#### 6.3.1 单选/多选

| 属性 | 规范 |
|------|------|
| 单选/多选框尺寸 | **16 × 16px** |
| 横向排列间距 | **16px** |
| 纵向排列间距 | **8px** |
| 单选框形状 | 圆形 |
| 复选框形状 | 方形 |

#### 6.3.2 选择器类型

系统支持以下选择器类型：

- 单选（Radio）
- 多选（Checkbox）
- 时间/日期选择器
- 开关切换
- 下拉选择
- 滑块选择
- 树选择
- 级联选择
- 穿梭框

### 6.4 表格

#### 6.4.1 行规范

| 属性 | 规范 |
|------|------|
| 行高 | **40px**（为字体高度的 2-3 倍） |
| 默认行数 | **10、20 或 50** 条/页，用户可按需选择 |
| 斑马纹 | 浅色行 `#FFFFFF`，深色行 `#FBFBFC`，表头 `#F6F6F6` |
| 悬停态 | `#1E63B2` 10% 透明度 |
| 选中态 | `#1E63B2` 10% 透明度 |

#### 6.4.2 列规范

| 属性 | 规范 |
|------|------|
| 列宽 | 内容固定时列宽大于固定宽度；不固定时按最大宽度或常用宽度设定 |
| 内容溢出 | 以 `...` 省略，鼠标悬停显示完整内容 |
| 列数过多 | 引入配置列，用下拉展开；可引入横向滚动条，固定首列和操作列 |
| 空值 | 直接留空 |
| 对齐 | 文本标题居中对齐、文本左对齐、数据右对齐、金额小数点对齐 |

#### 6.4.3 查看/编辑态

- **查看态**：以纯文本展示，无交互控件
- **编辑态**：输入框/选择器等可操作控件

#### 6.4.4 列操作

| 操作 | 说明 |
|------|------|
| 列排序 | 字段下拉框排序；选中后字段旁出现快速排序按钮 |
| 冻结列 | 字段下拉中选择冻结/取消冻结 |
| 拖动列 | 长按字段名或鼠标点击字段名左右拖动 |
| 手动改变列宽 | 鼠标悬停字段边界，拖动调整 |
| 提示说明 | 复杂字段的提示说明 |

#### 6.4.5 行操作

- 操作列位于右侧，操作按钮数 ≤ 3 个，多余内容放入"更多"菜单
- "查看"按钮放操作列首位，查看频率远高于编辑

### 6.5 选项卡

| 属性 | 规范 |
|------|------|
| 使用场景 | 同一模块内组织多个相同性质但分别独立的内容 |
| 功能 | 快速切换，起导航作用 |
| 可定制部分 | 滑动条颜色、文字链接颜色、按钮颜色、圆角、选中项底色 |

### 6.6 分页标签

适用于有大量内容需要分页加载处理的场景。组件包含页码、上下页、每页条数选择、快速跳转等元素。

### 6.7 弹窗

#### 6.7.1 基础规范

| 属性 | 规范 |
|------|------|
| 遮罩颜色 | `#000000`，不透明度 **60%** |
| 内容区域 | 宽高视业务而定 |
| 卡片圆角 | **2px** |
| 关闭方式 | 右上角关闭图标，确认/取消按钮 |

#### 6.7.2 使用场景

弹窗分为**内容弹窗**和**提示弹窗**：

- 内容弹窗：一级页面的高级查询、二级页面列表的常用操作、编辑/新增表单
- 提示弹窗：仅需一个按钮关闭的各类信息提示

#### 6.7.3 注意事项

- 弹窗是强反馈机制，仅在传递非常重要且可操作的信息时使用
- 除操作失败外，避免显示不必要的提醒弹窗
- 弹窗出现时用户不可进行任何其他操作
- **不建议模态弹窗再弹模态弹窗**

### 6.8 侧滑窗

#### 6.8.1 基础规范

| 属性 | 规范 |
|------|------|
| 遮罩颜色 | `#000000`，不透明度 **45%** |
| 按钮布局 | 主行动按钮居左，次级按钮居右 |
| 按钮跟随 | 内容超出一屏后，按钮固定于底端 |
| 层级限制 | 支持内容向下探一级（可使用二级侧滑窗或就地编辑） |

#### 6.8.2 宽度

| 尺寸 | 宽度 | 适配说明 |
|------|------|-----------|
| XS | **320px** | 最小宽度 |
| S | **480px** | 窄面板 |
| M | **640px** | 标准面板 |
| L | **800px** | 宽面板 |
| XL | **960px** | 最大宽度 |

> 屏幕 ≤ 1280px 时，宽度控制在 **600px** 内；屏幕 > 1280px 时自适应变宽，但不超过 **900px**。

#### 6.8.3 使用场景

- 列表设置
- 条件选择
- 意见处理
- 操作日志查看
- 选项修改

### 6.9 提示

#### 6.9.1 Toast（消息框）

| 属性 | 规范 |
|------|------|
| 位置 | 顶部居中 |
| 持续时间 | 一般 **3 秒**自动消失 |
| 使用场景 | 成功、警告、错误等反馈信息 |
| 设计原则 | 不打断用户当前操作 |

#### 6.9.2 警告提示

| 属性 | 规范 |
|------|------|
| 位置 | 页面顶部（或其他位置） |
| 持续时间 | **不会自动消失**，用户可点击关闭 |
| 使用场景 | 模块或功能发生重要改变，需持续提示用户 |
| 设计原则 | 非阻碍式信息展示，不打断用户当前操作 |

#### 6.9.3 全局提示（徽标）

出现在通知图标或头像右上角，显示需处理的消息条数，以醒目视觉形式吸引用户处理。

### 6.10 加载状态

| 等待时间 | 反馈方式 |
|----------|----------|
| < 0.1 秒 | 无需 loading 显示（用户感知为即时） |
| 0.1 – 1 秒 | 需要 loading 显示 |
| > 1 秒 | 建议告知真实进度百分比（完成度或等待时间） |
| > 10 秒 | 需告知进度且不建议阻挡用户做其他操作（非同步操作） |

### 6.11 进度条

- **标准进度条**：水平条状，显示完成百分比
- **圆形进度条**：适用于环形展示场景

### 6.12 树形组件

| 属性 | 规范 |
|------|------|
| 位置 | 列表内容区左侧 |
| 图标尺寸 | **16 × 16px** |
| 图标类型 | 可配置 |
| 交互 | 可收起、可展开、可拖拽 |

### 6.13 折叠面板

内容描述仅限一行显示，可展开/折叠。

### 6.14 步骤条

用于引导用户完成多步骤操作的流程指示器。

### 6.15 气泡提示

| 属性 | 规范 |
|------|------|
| 使用场景 | 解释功能、显示帮助文案等 |
| 触发方式 | 常在问号图标上触发 |
| 内容承载 | 可承载链接、按钮等复杂内容 |

### 6.16 数据图表

用于数据可视化展示，样式统一遵循品牌色体系。根据业务需要选择合适的图表类型（折线图、柱状图、饼图、散点图等），图表配色与功能色协调一致。

## 7 交互规范

### 7.1 信息获取（搜索）

| 规范项 | 说明 |
|--------|------|
| 搜索类型 | 支持文本、数字、标示码搜索（如单位名称、单位编码、社会统一信用代码） |
| 模糊查找 | 支持 |
| 高级查询 | 支持 |
| 批量操作 | 复杂业务可使用"搜索+选择"方式（如单位树中复选单位） |

### 7.2 信息处理（表单页）

| 规范项 | 说明 |
|--------|------|
| 默认填写 | 增加默认填写、自动填写（函数录入）和关联信息推荐 |
| 查找引导 | 优先引导用户搜索，采用"搜索+下拉菜单""搜索+弹窗入口"等复合组件 |
| 参考信息 | 支持两栏信息展示，支持读取往年信息等参考信息页面 |
| 跨表单复制 | 支持跨表单信息复制 |

### 7.3 信息反馈

完整的信息反馈体系包含三种形式：

| 反馈类型 | 组件 | 特点 |
|----------|------|------|
| 即时反馈 | Toast 消息框 | 轻量级，自动消失 |
| 持续提示 | 警告提示条 | 不打断操作，可手动关闭 |
| 阻断式反馈 | 弹窗提示 | 强反馈，需用户响应 |

其他反馈细节：

- **空状态**：当无内容或数据时，提供说明信息或操作指引
- **校验反馈**：输入框漏填/错填时在输入框下方展示错误提示
- **操作结果**：成功/失败通过 Toast 告知用户

## 8 视觉呈现

### 8.1 页面浏览顺序（F 型浏览模式）

用户浏览内容区块时遵循 F 型模式：

1. 首先在内容区上半部分水平方向阅读 —— 形成 F 型头部
2. 在屏幕左侧垂直浏览，寻找感兴趣的内容 —— 形成 F 型躯干
3. 再次水平快速浏览 —— 形成 F 型尾部
4. 最后在垂直方向浏览左侧区域

**设计启示**：用户最先关注右侧顶部操作区域和左侧标签栏，应在此优先展示核心操作和导航入口。

### 8.2 页面视觉层级（格式塔原理）

- 使用品牌蓝色标识重要信息，蓝色为主体，其他为背景
- 限制颜色种类使用，避免弱化主体与背景的边界
- 表格采用**斑马纹**（`#FFFFFF` / `#FBFBFC`）避免纵向视线错位

## 9 操作规范

### 9.1 弹窗操作

- 弹窗为**模态弹窗**，可移动、可缩放
- 模态背景色为浅色，方便用户看到底层页面内容
- 模态弹窗对应模态背景，**不建议模态弹窗再模态弹窗**

### 9.2 侧滑窗操作

- 为保持体验一致性，**不建议侧滑窗再弹模态弹窗**
- 侧滑窗内可使用的操作方式：
  - 就地编辑
  - 二级侧滑窗
