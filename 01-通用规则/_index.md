## 与 skills 的关系

这些规则主要不是给人读的，而是被 AI 执行 skill 时自动引用。例如：
- `prototype-html-pin` 画原型时会读 `设计库/` 和 `设计规范/`，并遵守 B 端原型 6 条准则
- `proto-spec-generator` 写规格时会引用 `方法论/原型设计规范.md`
- `prd-writer` agent 写 PRD 时以 `模板/PRD模板.md` 为骨架，完成后跑 `检查清单/需求评审检查清单.md`

所以这套规则的价值在于"被 AI 引用"而非"被人阅读"。修改这里的文件，下次 AI 执行对应 skill 时自动生效。这也是 PMCockpit 区别于普通 Prompt 模板集的核心：规则是结构化的、可被 skill 编程式引用的，而不是一段需要人手动粘贴的提示词。
