# 贡献指南

感谢你对 PRDCraft 的兴趣。这套框架的价值来自真实产品场景的打磨，欢迎贡献你的经验。

## 三种贡献方式

### 1. 提 issue（最轻，最欢迎）
使用中遇到任何问题，直接提 issue：
- skill 触发不稳定（说了关键词没反应）
- AI 产出不符合规范（格式、字段、详略出问题）
- 文档不清楚、步骤跑不通
- 想要某个新功能（比如"支持 Figma 导出"）

**好的 issue 包含**：你说了什么、AI 做了什么、你期望的是什么、用的哪个工具（ZCode/Claude/Codex）。

### 2. 改进现有 skill 或规则
发现某个 skill 的步骤可以优化、某条规则表述不清，欢迎提 PR：
- Fork 仓库 → 建分支 → 改 → 提 PR
- PR 描述写清楚：改了什么、为什么改、改前后效果对比

### 3. 贡献新 skill
如果你沉淀了可复用的工作流（比如"竞品分析自动化""数据指标字典生成"），可以贡献为新 skill。新 skill 需包含：
- `SKILL.md`：触发条件、工作流程、输出标准、参考路径（参照现有 skill 的结构）
- 必要的 assets（模板、脚本）
- README 说明（可选）

## 清洗规范（提 PR 前必读）

本框架面向开源，PR 内容**不得包含**：
- 真实公司名、客户名、项目名（用 `<公司名>` `<项目名>` 等占位符）
- 真实服务器 IP、密码、API key、SSH 密钥
- 真实个人路径（如 `/Users/xxx/...`），改用相对路径
- 真实人名、可识别的个人信息

提交前自查：`grep -rn "你的公司名\|真实IP\|真实人名" .` 确保无残留。

## 写作规范

贡献的 skill 或规则文档，遵守 AGENTS.md 的文档生成纪律，重点：
- 功能描述 67 到 120 字，阿拉伯数字分点
- 禁止 emoji、箭头符号（→、=>）
- 角色首次出现带边界说明
- 流程有完整闭环（触发条件、操作步骤、执行结果）
